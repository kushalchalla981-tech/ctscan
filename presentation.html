The Big Picture

From Measurements to Image

CT scanners measure X-ray attenuation from multiple angles. The challenge: recover the internal image from these projections — a classic linear inverse problem.

Given measurements A · x = b   →   Find x
🔵

The Forward Problem Easy

Given an image x, simulate the X-ray beams. Each ray's path through the pixels defines a row in matrix A. The result is a sinogram b — the measurements a real detector would record.

🔴

The Inverse Problem Hard

Given the sinogram b and system matrix A, recover the original image x. This is ill-posed: small measurement errors amplify into large reconstruction errors.

Key insight: The system is overdetermined (1,408 equations, 1,024 unknowns) and 91% sparse — most matrix entries are zero because each X-ray beam passes through only a fraction of pixels. This sparsity is the key to efficient solving.

Phase 2

The Mathematics of Solving A·x = b

▸ Least-Squares Solution

Since A is not square, we minimize the squared error:

x̂ = argmin ‖A·x − b‖²

The solution satisfies the normal equations: (Aᵀ·A)·x = Aᵀ·b

▸ Two Solver Strategies

Dense LU

Factor Aᵀ·A into P·L·U → forward & back substitution. Exact but expensive for large systems.

Sparse LSQR

Iterative method (Golub-Kahan bidiagonalization). Works directly with sparse A — no dense Aᵀ·A needed.

Iterative Refinement
Correction step
x̂ₖ₊₁ = x̂ₖ + A†·(b − A·x̂ₖ)
Auto-Select
Dense ↔ Sparse
Chooses best method based on size
Tikhonov Reg.
Noise dampening
min ‖A·x − b‖² + λ²‖x‖²

What the Metrics Mean

RMSE

Average pixel error. 0.022 = 2.2% error per pixel. With refinement: 0.12% — near perfect.

PSNR

Peak Signal-to-Noise Ratio. >30 dB is good; >50 dB is excellent. Refinement achieves 58.35 dB.

SSIM

Structural Similarity — compares patterns, not just pixels. 1.000 = structurally identical to ground truth.

SSIM = 1.000 with iterative refinement: the reconstruction is structurally indistinguishable from the original phantom when no noise is present.

Noise Robustness & Regularization

Real CT measurements are noisy. Without regularization, the inverse problem amplifies noise catastrophically.

0%0.08321.6 dB0.913

⚖️ Tikhonov Regularization

Instead of solving min ‖A·x − b‖², we solve:

min ‖A·x − b‖² + λ² · ‖x‖²

The second term penalizes large solutions, preventing noise amplification. Parameter λ (damp) controls the bias–variance tradeoff.

Without regularization: RMSE jumps from 0.022 → 2.56 at just 1% noise!
With regularization: RMSE stays at 0.095 — 27× better.

✓ What We Learned

1. Inverse problems are ill-posed — tiny errors become huge reconstruction failures.

2. Sparsity is a superpower — 91% sparse matrices make iterative solvers practical.

3. Regularization is essential — bias–variance tradeoff is central to all inverse problems.

4. Different tools for different jobs — LU for small/dense, LSQR for large/sparse.
